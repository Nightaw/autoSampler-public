# 项目摘要

如果把这个仓库当成面试展示材料，我会重点讲三件事：

## 1. 我不是只做“采样”

采样只是入口。  
真正更有工程含量的是采样之后的标签治理、证据生成、结果结构化和系统接入。

## 2. 我会把人工流程做成可复跑模块

标签初筛这件事原本很容易停留在“人工看一下”。  
我把它拆成了：

- 可复跑的规则评分
- 可复核的证据图
- 可落库的结构化结果

这样后续才能接平台、接队列、接统计。

## 3. 我重视“展示方式”本身

公开版不是把工作目录直接发出去，而是重新组织成：

- 有明确主线
- 有 API
- 有测试
- 有 CI
- 有文档

这说明我在意的不只是把功能做出来，也在意怎么让别人快速看懂。
